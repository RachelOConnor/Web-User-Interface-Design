Introduction to Web Architecture

Web architecture is a crucial part in the functioning of the World Wide Web. Web architecture allows data to be easily accessible via numbers, letters, comprehensible words, etc.

Web Servers: Physical vs. Virtual

Web servers use "http" to serve files that make pages to user browsers. These files, such as HTML files, image files, etc, are forwarded to the clients via http. Servers can run more than one service at a time, such as hosting sites, files, emails, and more.
Aspect Physical Server Virtual Server
Hardware Controlled by operating system - physical computer or digital cloud Application installed on server
Scalability Can run multiple services at once (files, email, host sites) Can host multiple sites at once
Cost Hardware can be expensive Cheaper - all digital

Web Protocols

HTTP:
HyperText Transfer Protocol

Facilitates data communication on the web by transferring web resources, such as HTML files, when requested by the client.

HTTPS:
HyperText Transfer Protocol Secure

Enhances the security of web communications by using encyrption to protect transferred data.

TCP:
Transmission Control Protocol

Allows for communication between the browser and the software to ensure requests are received and fulfilled.

IP:
Internet Protocol

Routes the packet to its destination, allowing the client to properly receive the requested data.

FTP:
File Transfer Protocol

Transfers files from a local machine to server.

URLs and DNS

URL: Uniform Resource Locator - a URL identifies resources available for the client to access.
A URL is made of multiple parts, shown below:

  • Protocol - http / https, etc - what is used to request and transfer resources.
  • Sub-domain - An extension to a website
  • Domain name - Identifiable name of website itself
  • Folder/Path/Resource - The section of the site or the file used to get to that page.

DNS: Domain Name System - translates readable names to IP address which allows the browser to locate domains by name rather than a numerical IP.

TLD: Top Level Domain - located at the last segment of a domain name. Can be a country code or other large body.
Examples:

  • .ie = Ireland's country code
  • .nl = Netherland's country code
  • .gov = Made by local government body

Cookies: Cookies are used to send saved data from the server to the client browser. The most common uses for cookies on the web is to remember a user's preferences, their location, and is sometimes even used to track a user's behaviour.

Browsers: Browsers are software applications that display content such as text, image, etc. using a render engine.


Comparing Browsers

Edge's web display is quite busy with popups, news articles, weather reports, etc all waiting to be clicked. This is due to their rendering engine and how they've decided to implement certain features.
When we compare Edge to Chrome, the comparison is like night and day.
Chrome has minimal features and provides a calmer, quieter space. Chrome also uses multiple rendering engines to drive the layout of their pages, as they feature animations which require things like javascript engines, etc.

Caching: Caching involves storing frequently accessed web resources.
For example, if you visit SETU's Student Portal often, the browser will store the files provided by this site. This allows the information to be recalled much quicker and reduces the load on the server.

Communication between Client and Server

Request-response Model: When the client sends a request, the server will respond with the requested data in a HTML format.
Stateless Protocol: HTTP is stateless, each request is independent and does not retain information - hence why cookies is used.
WebSockets: Allows for two-way communication and is used for things like real time chat and live updates.

Client-side Development Technologies

Client-side technology is what the client will see. This involves design, animations, links, etc.
This is done via HTML, CSS and Javascript.
HTML: Creates structure and controls the layout.
CSS: Styles elements and targets different screen sizes.
Javascript: Increases interactivity and handles complex features.
